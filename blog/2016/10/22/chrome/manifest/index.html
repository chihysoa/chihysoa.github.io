每一个插件、可安装的WebApp、皮肤，都有一个JSON格式的manifest文件，叫manifest.json，里面提供了重要的信息。他有以下字段

name,version,manifest_version,description,icons,default_locale,browser_action,page_action,theme,app,background,chrome_url_overrides,content_scripts, content_security_policy,file_browser_handlers,homepage_url,incognito,intents,key,minimum_chrome_version,nacl_modules,offline_enabled,omnibox,options_page,permissions,plugins,requirements,update_url,web_accessible_resources

name

必选字段，插件的名称，字符串类型

version

必选字段，插件的版本号，字符串类型，版本号，一个到4个数字来表示，中间用点隔开。这些数字有些规则：必须在0到65535之间，非零数字不能0开头。

manifest_version

必选字段，manifest文件自身格式的版本号，数字，目前manifest_version为2

description

非必选字段，建议添加，插件的描述，字符串类型，只能纯文本，不能是html等。

icons

非必选字段，建议添加，插件的图标，可以给定一个或多个

default_locale

非必选字段，指定这个插件保的缺省字符串的子目录，如果插件有_locales目录，这个字段是必须的。如果没有_locales目录，这个字段是必须不存在的。

browser_action

非必选字段，用于配置插件界面，browser_action，page_action，theme，app四者选一或都不选。

page_action

非必选字段，用于配置插件界面，browser_action，page_action，theme，app四者选一或都不选。

theme

非必选字段，用于配置主题，包括图片，颜色等，browser_action，page_action，theme，app四者选一或都不选。

app

非必选字段，用于配置可安装的webapp，包括打包过的app，需要这个字段来指定app需要使用的url。最重要的是app的启动页面——当用户在点击app的图标后，浏览器将导航到的地方，browser_action，page_action，theme，app四者选一或都不选。

background

非必选字段，用于配置背景页，包括js，css等。

chrome_url_overrides

非必选字段，用于配置使用替代页，可以将Chrome默认的一些特定页面替换掉，改为使用插件提供的页面。可选值为：pageToOverride，bookmarks，history，newtab。

content_scripts

非必选字段，用于配置在Web页面内运行的javascript脚本。

content_security_policy

非必选字段，用于配置CSP策略，默认规则：script-src 'self'; object-src 'self'(这条规则将同时禁止内嵌script块和内联事件)，更加详细的配置可以参考CSP文档

file_browser_handlers

homepage_url

非必选字段，用于配置插件的主页，缺省就是插件的页面。

incognito

非必选字段，用于配置插件在允许隐身模式下运行时如何响应。可选值：”spanning”和”split”，缺省值是split。

intents

非必选字段，一个字典，用于描述插件或app所提供的全部intent handler。字典里的每个键指定了一个action verb。

key

非必选字段，开发时为插件指定的唯一标识值。代码中使用相对路径或者chrome.extension.getURL()得到的绝对路径。

minimum_chrome_version

非必选字段，指定chrome的最小版本。

nacl_modules

非必选字段，一个或多个从MIME到处理这个MIME的本地客户端模块之间的映射。

offline_enabled

非必选字段，用于指定本插件或app是否支持脱机运行。当Chrome检测到处于脱机状态，此项设置为“是”的app将会在新标签页中高亮显示。

omnibox

非必选字段，omnibox 应用程序。程序界面允许向Google Chrome的地址栏注册一个关键字，地址栏也叫omnibox。使用omnibox 应用程序界面，必须在 manifest 中包含omnibox 关键字段。需要指定像素为16x16的图标，以便当用户输入关键字时，在地址栏中显示。

options_page

非必选字段，为了让用户设定你的插件功能，你可能需要提供一个选项页。如果你提供了选项页，在插件管理页面 chrome://extensions上会提供一个链接。点击选项链接就可以打开你的选项页。

permissions

非必选字段，插件或app将使用的一组权限。每个权限是一列已知字符串列表中的一个，如geolocatioin或者一个匹配模式，来指定可以访问的一个或者多个主机。权限可以帮助限定危险，如果你的插件或者app被攻击。一些权限在安装之前，会告知用户。

plugins

非必选字段，NPAPI 插件，使JavaScript代码能够调用本地二进制代码，运行在NPAPI插件中的代码拥有当前用户的全部权限，不能利用Google Chrome 的沙箱技术和其他安全防护技术。 在处理不可信任的输入， 如content scripts和XMLHttpRequest 时，你必须格外小心。

“path” 属性用于描述如何找到你的插件，路径是相对于manifest文件位置的。 “public” 属性指明是否允许普通页面加载你的插件，默认是false，也就是只有你的插件才能加载这个插件。
创建一个HTML文件，mime-type为：application/x-my-extension” ，用于加载你的插件。

这个页面可以被后台页面包含，或者任何你插件会用到的其他页面。如果你的插件的 “public” 属性是true，，你可以通过脚本在页面中直接使用它。

requirements

非必选字段，指定本app或插件所需的特殊技术功能。安装插件时，插件商店根据这个清单，必要时劝阻用户在不支持所需功能的电脑上安装这些插件。

update_url

必选字段，一个插件的manifest文件里面必须指定一个”update_url”来执行升级检测。
插件可以托管在Chrome Web Store，也可以发布到极速浏览器应用开放平台上。
如果托管在Chrome Web Store则update_url应该是：http://clients2.google.com/service/update2/crx
如果托管在极速浏览器应用开放平台上则update_url应该是：http://upext.chrome.360.cn/intf.php?method=ExtUpdate.query

web_accessible_resources

非必选字段，一组字符串，指定本插件在注入的目标页面上所需使用的资源的路径（相对于插件的安装根目录）。例如，插件在example.com上注入脚本以构建制界面，将其间所需的资源（图片、图标、样式、脚本等）加入白名。
