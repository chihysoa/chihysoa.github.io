最近在搞chrome插件开发，为了方便调试就顺便搞了个请求拦截的插件
关于chrome插件对网络请求的处理用到chrome.webRequest模块，下面是WebRequest API

权限

Manifest
在manifest.json里面添加权限

事件

onBeforeRequest

请求准备之前时触发

onBeforeSendHeaders

请求准备完成,初始化请求头信息时触发

onSendHeaders

在请求头发送之前触发

onHeadersReceived

在响应头接收之后触发

onAuthRequired

当请求需要用户认证时触发

onBeforeRedirect

在一个请求重定向发起之前触发

onResponseStarted

当响应体开始下载之前触发

onCompleted

请求结束时触发

onErrorOccurred

请求失败时触发

onComplete或者onErrorOccurred是最后的事件，并且只会触发其中的一个

使用

用chrome.webRequest.事件.addListener()方式来添加监控
addListener方法有三个参数

参数一

一个函数，有一个对象，包含有url，header等属性
返回一个BlockingResponse对象，具有以下属性
cancel：取消
redirectUrl:重定向url
requestHeaders:请求头
responseHeaders:响应头
authCredentials:认证，是个对象，包含username和password

参数二

一个filter拦截列表，用来定义需要拦截的请求

参数三

一个数组，为一组规范，一般为blocking,requestHeaders
