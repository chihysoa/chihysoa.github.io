webpack-dev-server是一个小型的node伺服文件服务器，在使用webpack构建的时候通常都会选用他。
安装方式通过npm来安装npm install webpack-dev-server --save-dev。

先看一段基本的webpack的配置

配置文件提供一个入口和一个出口， webpack根据这个来进行打包和编译的工作。
webpack-dev-server有两种使用方式：命令行和node服务。
主要介绍一下命令行的使用
下面是一个项目结构

我们在命令行执行webpack构建时，会将src/main.js打包到dist下。
现在我们使用webpack-dev-server命令来构建看看结果

需要注意的是webpack.config.js配置文件中output下的path必须是绝对路径。

构建完毕会发现，并没有在dist下生成bundle.js这个文件
我们在浏览器里输入http://127.0.0.1:8080会发现打开了我们项目根目录下的index.html这个文件。
我们编辑一下这个index.html这个文件，引入bundle.js

这里要注意，我们要引入的bundle.js的路径和webpack的配置文件里面output的publicPath是一致的。
编辑完，我们再看下浏览器，发现bundle.js文件是存在的，并且成功引入了，是的，webpack-dev-server构建出来的文件都是在内存里面的，不会生成实际的文件。
webpack-dev-server还有其他设置

content-base

--content-base用来设置webpack-dev-server伺服的文件夹根目录，默认为当前目录，当访问http://127.0.0.1:8080时，会从所配置的目录查找文件。

hot

--hotwebpack-dev-server热部署，自动刷新功能

progress

--progress显示打包速度

port

--port 端口号指定webpack-dev-server的端口

iframe和inline模式

webpack-dev-server提供两种页面的模式，一种是inline整个页面，另外一种是iframe使用iframe来嵌套页面，我们访问http://127.0.0.1:8080时候就是inline模式，iframe模式通过http://127.0.0.1:8080/webpack-dev-server/index.html来访问，这里index.html就不能省略了
iframe的好处是，你可以看到变化的信息。

还有一些不常用的配置

  • --colors 设置终端字体颜色
  • --quiet 控制台中不输出打包的信息
  • --compress 开启gzip压缩
