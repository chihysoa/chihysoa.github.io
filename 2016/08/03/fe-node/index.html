前后端分离方案 - NodeJs

  • 前端
前端

什么是NodeJs

Nodejs也是这几年非常火爆的技术，它的社区非常活跃，而且采用的是chrome v8引擎，也就是说它用的是js语言，就是我们前端所用的js语言，所以由我们前端来写再合适不过了。

NodeJs的角色

这里提一下传统的开发模式，在传统的开发模式下数据服务提供者和模板渲染都是后端服务器，Nodejs承担的角色就是把数据服务和模板渲染给分开，数据服务还是由后端提供，但是模板渲染则交给Nodejs去处理。
和spa也有一定的区别，首先由于是服务器渲染，这种方式可以完美的支持SEO，但是在用户体检方面spa占优。其次虽然NodeJs是使用的是js语言，但是用NodeJs作为服务器对开发者的要求会比spa的要求要高，需要开发者具备一定的服务器端的专业知识而且对于NodeJs服务器的稳定性，健壮性，性能方面都会有一定的要求。

项目结构

项目划分：

  • 后端
    提供数据接口服务
  • 前端
    NodeJs 包括路由，模板引擎页面渲染，与后端通信
    js 各个页面的逻辑，也可以直接与后端通信（Ajax）
    css 控制页面样式

流程控制


当用户在浏览器打开一个页面，首先会到nginx，nginx转到node，node通过所配置的路由找到相应的逻辑处理，请求后端数据服务，通过模板引擎渲染页面，返回html代码。
浏览器获取html代码后开始解析，并加载css和js文件，这里的css和js文件就是静态资源。
由于NodeJs的代码也是js写的，html里面使用的也是js语言，这里大家要注意区分，这两个js并不是同一种东西，他们是两个完全独立的，只是恰好使用的都是js语言而已。
