这里不对单页应用的概念进行阐述了，如果有不清楚的请自行百度或谷歌。
既然是单页应用，前端入口只有一个html以及若干js和css资源文件。

项目划分：

  • 后端
    提供数据接口服务
  • 前端
    html 入口
    js 控制整个单页应用的逻辑，包括路由，页面渲染，与后端通信
    css 控制页面样式

前端与后端的通信主要通过ajax，下面是一个简单的项目结构（前端代码）

  • index.html 是我们spa的入口文件
  • static文件夹下面是我们的静态资源文件，像js，css，图片等

因为是spa，所以页面的路由掌控在我们前端自己的手里，后端为我们提供的只是一个个数据接口的服务，所以只要我们能够自己模拟实现这个数据接口，我们就可以完全脱离后端去开发。
首先，我们先不依托于数据接口，先把页面展示，逻辑都跑起来。这个就比较简单了，我们把前端代码随便丢到一个服务器容器里面就可以，比如nodejs，tomcat，nginx等。这里我比较推荐nginx，下面我也会使用nginx作为服务器。

当然spa有它的优缺点：

  • 因为是单页所以第一次打开的时候需要加载的资源比较多（这个也可以通过其他手段优化，比如资源懒加载 按需加载等），第一个页面会比较慢，但是之后用户体验较好
  • 页面全是通过js去控制路由，渲染，所以不支持SEO（解决方法也有很多，比如单独放一套供SEO使用的页面快照）
  • 对服务器的压力会比较小
  • 由于业务逻辑都在js，所以对开发人员的js要求会稍微高点
